# vue-cli3中proxy配置不完全支持webapck

# 结论

  1. 目前vue-cli3只支持string和object方式，不支持context数组方式。相关issue
  • Add more options for proxy context
  • 增强proxy: 对齐webpack devServer的proxy

# webpack proxy

官方文档 webpack-dev-server代理是基于http-proxy-middleware，更多使用方式可以查看其github

# 一

请求到 /api/xxx 现在会被代理到请求 http://localhost:3000/api/xxx

mmodule.exports = {
    //...
    devServer: {
        proxy: {
            '/api': 'http://localhost:3000'
        }
    }
};

# 二

代码多个路径代理到同一个target下, 你可以使用由一个或多个「具有 context 属性的对象」构成的数组

module.exports = {
    //...
    devServer: {
        proxy: [{
            context: ['/auth', '/api'],
            target: 'http://localhost:3000',
        }]
    }
};

# 三

传递参数

module.exports = {
    //...
    devServer: {
        proxy: {
            '/api': {
                target: 'http://localhost:3000',
                changeOrigin: true,
                secure: false,
                pathRewrite: {'^/api' : ''},
                bypass: function() {...}
            }
        }
    }
};
